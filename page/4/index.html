Git工作流程

发表于 2017-06-12 | 分类于 JAVA | | 阅读次数

Git工作流程

一般工作流程如下：

  • 克隆 Git 资源作为工作目录。
  • 在克隆的资源上添加或修改文件。
  • 如果其他人修改了，你可以更新资源。
  • 在提交前查看修改。
  • 提交修改。
  • 在修改完成后，如果发现错误，可以撤回提交并再次修改并提交。

##Git 工作区、暂存区和版本库

  • 工作区：就是你在电脑里能看到的目录。
  • 暂存区：一般存放在 “.git目录下”下的index文件（.git/index）中，所以我们把暂存区有时也叫作索引（index）。
  • 版本库：工作区有一个隐藏目录.git，这个不算工作区，而是Git的版本库。
    • 执行 “git add” 命令时，暂存区的目录树被更新，同时工作区修改（或新增）的文件内容被写入到对象库中的一个新的对象中，而该对象的ID被记录在暂存区的文件索引中。
    • 执行提交操作（git commit）时，暂存区的目录树写到版本库（对象库）中，master 分支会做相应的更新。
    • 执行 “git reset HEAD” 命令时，暂存区的目录树会被重写，被 master 分支指向的目录树所替换，但是工作区不受影响。
    • 执行 “git rm –cached “ 命令时，会直接从暂存区删除文件，工作区则不做出改变。
    • 执行 “git checkout .” 或者 “git checkout – “ 命令时，会用暂存区全部或指定的文件替换工作区的文件。这个操作很危险，会清除工作区中未添加到暂存区的改动。
    • 执行 “git checkout HEAD .” 或者 “git checkout HEAD “ 命令时，会用 HEAD 指向的 master 分支中的全部或者部分文件替换暂存区和以及工作区中的文件。这个命令也是极具危险性的.

Git官方文档

Git使用指南

git rebase 和 git merge

  1. 创建仓库：git init
  2. 克隆远程代码：git clone -b <分支名> <代码库地址>
  3. 添加到暂存区：git add 或者git add -A(add全部修改)
  4. 提交到本地库：git commit -m “tag”
  5. 提交到远程库：git push origin <本地分支>:<远程分支>
  6. 连接远程服务器：git remote add origin <远程服务器地址>
  7. 分支管理
    • 新建分支：git branch <分支名> 或者 git checkout -b <分支名>
    • 切换分支：git checkout <分支名>
    • 删除本地分支：git branch -d(-D) <分支名>（-D：强制删除）
    • 删除远程分支：git push origin –delete <分支名> 或者 git push origin :<分支名>（推送一个空分支）
    • 查看本地分支：git branch
    • 查看所有分支：git branch -a
    • 查看本地分支和远程分支的关联情况：git branch -vv
    • 合并分支：git merge <分支名> 或者 git rebase <分支名>
  8. 获取最新改动：git pull 或者 git fetch 或者 git pull <远程库名> <远程分支名>:<本地分支名>

  9. 删除修改：

    • 未add到暂存区：git checkout – 或者 git checkout .
    • 已add到暂存区：git reset HEAD 然后再执行未add的暂存区的步骤

    • 已commit到本地库：git reset –hard 或者 git reset –hard HEAD^(有几个^就回退几个版本)

      git revert 是生成一个新的提交来撤销某次提交，此次提交之前的commit都会被保留
      git reset 是回到某次提交，提交及之前的commit都会被保留，但是此次之后的修改都会被退回到暂存区

  10. stash
    • 暂存本地修改：git stash
    • 查看暂存：git stash list
    • 恢复暂存：git stash apply 或者 git stash pop（恢复最近一次暂存）
    • 清除暂存：git stash clear
  11. tag
    • 打标签：git tag -a -m “注释”
    • 查看标签：git tag 或者 git tag -l
    • 删除标签：git tag -d
    • 提交标签到远程库：git push origin 或者 git push origin –tags
    • 删除远程标签：git push origin :<远程tagName>
    • 切换到某个标签代码：git checkout -b <分支名>

项目初始化

  • 创建项目
  1. mkdir test – 创建本地文件夹
  2. cd test – 进入到本地文件夹
  3. git init – 初始化本地git库
  4. touch README.md – 添加本地文件
  5. git add README.md – add到缓存区（git add -A add全部修改）
  6. git commit -m “first commit” – 提交到本地git库
  7. git remote add origin http://192.168.0.110/test.git – 关联远程git库地址
  8. git push -u origin master – 提交到远程分支
  • 添加已有项目
  1. cd existing_git_repo
  2. git remote add origin http://192.168.0.110/test.git
  3. git push -u origin master
  • 拉取已有项目
    git clone -b master http://192.168.0.110/test.git

  1. 上面部分为远程分支，下面部分为本地分支

  4. 分支合并均在本地merge后再提交到远程分支，不允许直接合并远程分支
  5. 纵向“变基”使用git rebase，横向合并分支使用git merge，切记！切记！

一.Activity的四种状态

  • Running 一个新的Activity启动入栈后，它在屏幕最前端，处于栈的最顶端，此时它处于可见并可和用户交互的激活状态。
  • Paused 当Activity被另一个透明或者Dialog样式的Activity覆盖时的状态。此时它依然与窗口管理器保持连接，系统继续维护其内部状态，它仍然可见，但它已经失去了焦点，故不可与用户交互。
